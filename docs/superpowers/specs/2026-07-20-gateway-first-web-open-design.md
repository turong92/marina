# Gateway-first web open design

## Goal

Marina에서 서비스를 브라우저로 여는 기본 경로를 안정적인 게이트웨이 origin으로 통일한다. Docker 서비스가 재기동되어 임시 호스트 포트가 바뀌어도 URL, 쿠키, 로그인 세션이 유지되어야 한다.

## URL policy

- 공통 URL 선택 함수는 `gatewayUrlFor(session, service)`가 반환하는 게이트웨이 URL을 가장 먼저 선택한다.
- 게이트웨이가 비활성화되었거나 Caddy를 사용할 수 없을 때만 실행 중 서비스의 `http://localhost:<published-port>/`를 fallback으로 사용한다.
- 임시 호스트 포트는 서비스 메뉴의 `호스트 포트로 열기` 보조 액션에 계속 노출한다. 사용자가 불안정한 origin임을 알고 명시적으로 선택하는 경로다.
- 게이트웨이가 정상인 동안 기본 열기 동작에서 호스트 포트 URL을 사용하지 않는다.

## Surfaces

- 로그 화면 헤더의 `openWeb` 버튼은 선택한 세션의 web 서비스에 공통 URL 선택 정책을 적용한다.
- 세션 카드의 대표 URL은 기존 게이트웨이 URL을 유지한다.
- 서비스 행 메뉴의 첫 번째 열기 액션은 공통 URL 선택 정책을 사용한다. 직접 포트 열기는 별도 보조 액션으로 유지한다.
- 연결 탭의 게이트웨이 노드는 기존 고정 도메인 동작을 유지하고, 호스트 포트는 복사 가능한 진단 정보로만 남긴다.

## UI behavior

- 로그 헤더 버튼의 툴팁은 실제 대상에 따라 `게이트웨이로 열기` 또는 `임시 호스트 포트로 열기`를 표시한다.
- 열 수 있는 URL이 없으면 버튼을 숨긴다.
- 새 탭으로 열 때 `noopener`를 사용한다.

## Testing

- 게이트웨이가 정상이고 web 포트가 함께 존재할 때 로그 헤더가 게이트웨이 URL을 선택하는 테스트를 추가한다.
- 게이트웨이를 사용할 수 없을 때만 호스트 포트로 fallback하는 테스트를 추가한다.
- 서비스 메뉴에 게이트웨이 기본 열기와 명시적 호스트 포트 보조 열기가 함께 유지되는지 검증한다.
- 실제 대시보드에서 로그 서비스 선택 후 `openWeb` 대상과 툴팁을 브라우저로 확인한다.

## Non-goals

- 게이트웨이 도메인 규칙, Caddy 라우팅, 포트 자동 할당 방식은 변경하지 않는다.
- 외부 공개 도메인이나 원격 접근용 인증 정책은 이번 변경에 포함하지 않는다.
