# 터미널 탭 멀티세션 + 분할 (cmux 化)

날짜: 2026-07-15
상태: 설계 확정 (형 승인) — 구현 대기, 브랜치 claude/marina-terminal-multi-tab-7757c2
선행: 연결 탭 재설계 + compose 워크벤치 UX main 배포 완료(23e49ba, 109/109)
시안: `scratchpad/term-mockup.html`(A안 가로 스트립 vs B안 세로 사이드바), `scratchpad/term-split.html`(B안 + 4분할)

## 배경·목표

- 현행 터미널 탭(app-10-term.js)은 **터미널이 딱 1개**다. 모듈 전역 `termInst/termTid/termSse/termRoot/termAgent`
  가 한 벌뿐이라, 워크트리를 `<select>` 로 바꾸면 기존 터미널을 dispose 하고 새로 연다.
- 백엔드는 워크트리당 세션을 재사용해 스크롤백은 살아있지만 **화면엔 한 번에 하나만** 보인다.
- 형 요구: **cmux 처럼 여러 셸을 동시에 열어두고 전환**, 그리고 **4분할**.
- 목표: 세션 N개 동시 운용 + 최대 4칸 분할 + 새로고침 복원, 고아 PTY 없음.

## 비목표

- tmux 식 무한 재귀 분할(3분할·6분할 비대칭 레이아웃) — 프리셋 4종으로 충분, 대시보드 pane 에선 4칸이 실용 상한
- 세션 이름 직접 변경(rename) — 자동 `셸 N` 라벨로 충분
- 터미널 검색·복사 툴바·링크 클릭 등 xterm 애드온 확장
- 프로젝트 전환 시 세션 필터링(D7 참조 — 의도적 제외)

## 확정 결정

### D1. 개념 3분할

현행 코드가 아래 셋을 전역변수 한 벌에 뭉쳐놓은 것이 단일 터미널의 근본 원인이다.

| 개념 | 정체 | 진실의 출처 |
|---|---|---|
| **세션** | 백엔드 PTY, `tid` 로 식별 | 백엔드 (`/api/term-list`) |
| **인스턴스** | 세션당 xterm 하나, 칸에서 빠져도 살아있음 | 프론트 메모리 |
| **칸(slot)** | 그리드 1~4칸, "어느 세션을 그리나"만 지정 | localStorage |

### D2. 백엔드 — 셸은 매번 새 세션, 에이전트는 재사용

- `_by_root` 재사용 계약을 **셸에서 제거** → 같은 워크트리에 셸 N개.
- **에이전트 attach 는 현행 유지** (`cwd::agent:<source>:<sid>` 키로 재사용). CC/CX 세션에 resume 을
  두 번 붙이면 안 되므로 이 재사용은 의도된 것이다.
- `term_open` 반환은 `{tid, reused}` 유지 — 셸은 항상 `reused: false`.

### D3. 백엔드 — `/api/term-list`

```
GET /api/term-list → {"sessions": [{tid, root, agent: {source,sid}|null, created, alive, fg, cmd, preview}]}
```

- 새로고침 복원의 근거이자 **고아 PTY 방지책**: 슬롯 도입으로 "경로=키" 자동 복구가 깨지므로,
  프론트가 tid 를 잊으면 살아있는 PTY 에 다시 붙을 방법이 없어져 6시간 TTL 까지 좀비로 남는다.
- 오프셋은 여기 싣지 않는다. `from` 의 기준값은 SSE 이벤트의 `off` 로 프론트가 누적하므로
  term-list 에 넣으면 쓰이지 않는 필드가 된다.
- **라벨은 "지금 뭐가 돌고 있나"** (형: "셸 1 말고 뭔가 알아볼 수 있을만한거로") — tmux 가 창 이름을
  실행 중인 프로그램으로 짓는 것과 같다. term-list 가 두 필드를 더 싣는다:
  - **`fg`** — 포그라운드 프로세스 그룹의 명령(`os.tcgetpgrp` → `ps`). 셸 자신이면 빈 문자열.
  - **`cmd`** — 마지막으로 친 명령. **스크롤백에서 파싱하지 않는다** — zsh ZLE 가 좁은 칸에서 명령을
    CR·EL 로 다시 그려서 어느 한 줄에도 원문이 없다(45칼럼 실측: `npm run build` → `npm run b`+`uild`
    두 줄). 복원하려면 에뮬레이터를 새로 써야 한다. 대신 **형이 친 바이트가 term_input 으로 그대로
    들어오니** 거기서 잡는다(엔터에서 확정, BS·Ctrl-C·ESC 시퀀스 처리). 한계: ↑ 히스토리는 못 따라가
    마지막으로 **친** 명령이 남는다 — 틀린 이름을 지어내는 것보다 낫고 실행 중이면 fg 가 정확하다.
  - 이름 우선순위: `fg`(지금 도는 것) → `cmd`(마지막으로 친 것) → `셸 N`(아무것도 안 친 새 셸뿐).
  - **`preview`** — 마지막으로 **의미 있는** 출력 줄(사이드바 부제). 그냥 마지막 줄을 집으면 유휴 세션은
    늘 빈 프롬프트라 아무것도 못 알려주므로, 눈이 하는 대로 끝의 빈 프롬프트는 건너뛴다. 명령을 친 줄에선
    프롬프트 접두사를 뗀다(안 떼면 200px 안에서 잘려 호스트명만 보인다). 백스페이스는 **적용**한다
    (zsh 자동완성이 `s\bsleep` 처럼 쓰고 지워서, 그냥 제거하면 `ssleep` 이 된다).
    **`redact_text` 필수** — 셸 스크롤백엔 비밀값이 섞이고 사이드바는 늘 떠 있다.
  - 이름과 부제가 같으면(출력이 아직 없는 명령 — `sleep 90` 은 마지막 줄이 방금 친 그 명령) 부제를 생략한다.
  - 에이전트 세션은 여전히 CC/CX 칩 + 세션 제목(`worktreeData.agents` 에서 sid 로 조회).
  - **신선도**: 뷰가 터미널 탭이 **보일 때만** 3초 폴링(`fg` 는 세션마다 `ps` 를 부른다). `connect()` 의
    no-op 가드가 없으면 이 폴링이 3초마다 SSE 를 끊는다.

### D4. 백엔드 — 멀티플렉스 SSE

```
GET /api/term-stream?tid=a,b,c&from=a:1024,b:0
  event: snap → data: {"tid","b64","off"}      # 스크롤백 스냅샷
  event: out  → data: {"tid","b64","off"}      # 신규 청크
  event: exit → data: {"tid"}
```

- **이유**: 브라우저는 HTTP/1.x 에서 오리진당 커넥션 6개. 서버가 `protocol_version` 미설정이라
  HTTP/1.0 이고, SSE 는 슬롯을 계속 점유한다. 탭마다 SSE 를 붙이면 **터미널 5개 + 로그 SSE 1개 = 6** 에서
  이후 `fetch` 가 전부 대기 → 키 입력(`/api/term-input`)도 fetch 라 **타이핑이 먹통**이 된다.
  멀티플렉싱하면 탭 수와 무관하게 커넥션 1개.
- **구현**: `_Term.cond` 를 **전 세션 공유 Condition 하나**로 교체(`self.cond = _COND`). 아무 세션의
  `append` 에나 깨어나므로 여러 tid 를 한 루프에서 감시할 수 있다. 기존 `with term.cond:` 사용부와
  테스트는 그대로 동작한다(락 순서 문제도 없음 — 조건변수가 하나뿐이라 중첩 획득 자체가 없다).
- **갭 처리**: `from` 이 링버퍼 `base` 보다 오래되면(출력이 밀려 잘림) 그 tid 만 `snap` 재전송.
  `from` 이 없으면 `snap`, 있고 유효하면 `history[from-base:]` 부터.
- 하위호환: `tid=a` 단일도 같은 경로로 동작(이벤트에 tid 태그가 붙는 것만 다름).

### D5. 프론트 — 파일 분리

현행 160줄에 사이드바·그리드·D&D·스트림·입력큐를 다 넣으면 400줄을 넘겨 한 파일이 두 일을 한다.

- **`app-10-term.js`** — 뷰: 사이드바 렌더, 그리드/분할, D&D, 칸↔세션 배치, WS_VIEWS 등록.
- **`app-10b-term-io.js`** — io: 세션 스토어(term-list), 멀티플렉스 SSE, tid별 입력큐, term-open/kill.
- 경계: io 는 tid 만 알고 DOM 을 모른다. 뷰는 칸만 알고 HTTP 를 모른다.
- index.html 로드 순서: `vendor-xterm*` → `app-10b-term-io.js` → `app-10-term.js` → `app-7-init.js`.

### D6. 프론트 — 입력큐·구독·인스턴스 수명

- **입력큐는 tid 별로**: 현행 전역 `termSendChain`/`termSendBuf` 한 벌 → `Map<tid, {chain, buf}>`.
  나누지 않으면 탭 A 의 글자가 탭 B 로 샌다. 직렬 큐 + 코얼레싱 성질은 tid 별로 그대로 유지
  (병렬 fetch 는 추월/유실로 글자가 사라진다).
- **구독 = 살아있는 세션 전부** — 커넥션이 1개이므로 공짜다.
- **인스턴스 생성 = 처음 칸에 배치될 때.** 그 시점에 해당 tid 만 `from` 없이 재연결해 `snap` 을 받는다
  (xterm 은 attach 된 요소에 `open()` 해야 치수가 잡히므로 미리 만들 수 없다).
- **인스턴스 파기 = 죽었고 + 칸에도 안 떠 있을 때**(에러 처리 절 참조). 살아있는 세션은 칸에서 빼도
  인스턴스·구독을 유지 → 재진입이 **재생 없이 즉시**, 스크롤 위치도 보존. 인스턴스 없는 세션은 청크를
  버리고 **활동 닷**만 켠다.
- **활동 닷**은 인스턴스 유무와 무관하게 한 규칙으로: **칸에 안 떠 있는 세션에 출력이 도착하면 켜지고,
  그 세션이 칸에 배치되면 꺼진다.** (인스턴스가 있어도 칸에서 빠져 있으면 형은 그 출력을 못 본 것이므로
  닷이 필요하다. 인스턴스 유무로 규칙을 가르면 이 경우가 구멍이 된다.)
- 스트림 재연결 조건: 구독 tid 집합 변경, 또는 어떤 tid 에 snap 이 필요할 때. 나머지 tid 는 `from` 으로
  이어받아 중복·유실이 없다.
- **테마**: MutationObserver 가 인스턴스 전부 순회(현행은 `termInst` 하나만 갱신).
- **리사이즈**: 칸마다 FitAddon + ResizeObserver → `term-resize(tid)`.

### D7. UI — B안(좌측 세로 사이드바) + 프리셋 분할

- **사이드바 200px(드래그로 조절, 더블클릭=기본 폭, localStorage 기억 — 깃 탭 gs-rail 과 같은 패턴)**: 워크트리별 그룹 헤더 + 세션 항목(`셸 1`/`셸 2`/CC·CX 칩+세션제목), **맨 위** `＋ 새 셸`(하단 고정은 목록이 짧을 때 손이 안 닿는다 — 형 지적).
  - 가로 스트립(A안)을 버린 이유: 상단 워크스페이스 탭줄(로그·깃·연결·터미널) 바로 밑에 또 가로 탭줄이
    생겨 두 겹으로 쌓이고, "같은 워크트리 N개" 라는 이번 모델의 묶음 단위를 이름 접두사로 흉내내야 한다.
  - 폭 비용은 감당 가능: 1600px 화면에서 ws-pane ~1200px − 200 = ~1000px → 12.5px 모노스페이스 기준 약 130칼럼.
- **분할 프리셋 4종**: `1` / `좌우2` / `상하2` / `4`. CSS Grid `grid-template-{columns,rows}` 로 표현.
  ~~칸 사이 경계는 드래그로 비율 조절.~~ → **미구현**(프리셋 4종으로 충분해 보류. 별도 상태(ratios)와
  핸들 DOM 이 붙는다 — 써보고 필요하면 후속. **형 확인 대기**).
- **배치**: 사이드바 항목 클릭 = 활성 칸에 배치. 항목을 칸으로 **D&D** = 그 칸에 배치.
  **활성 칸**은 마지막으로 클릭·포커스한 칸이고, 초기값·복원 실패 시 1번 칸이다.
- **`＋ 새 셸`** 은 워크트리 선택 드롭다운을 연다(현행 `<select data-term-wt>` 의 라벨·중복 해소 로직 재사용).
  현재 프로젝트의 워크트리를 위에, 나머지를 아래에 둔다. 고르면 `term-open` → 활성 칸에 배치.
- **표시**: 사이드바 항목의 **①②③④ 배지** = "이 세션이 몇 번 칸에 떠 있나"(배지 없음 = 살아만 있고 안 보임).
  칸 파란 테두리 = 키 입력이 가는 칸. 활동 닷 = 안 보이는 세션에 새 출력.
- **프로젝트 전환 추종 안 함**(의도적): 연결 탭은 follow 하지만, 여기서 필터링하면 dev 서버가 도는 세션이
  프로젝트를 바꿨다는 이유로 사라져 위험하다. 사이드바는 전역으로 전부 보이고, `＋ 새 셸` 의 워크트리
  후보만 현재 프로젝트를 우선 제시한다.

### D8. 닫기 — 두 ✕ 의 의미가 다르다

| 위치 | 동작 | 결과 |
|---|---|---|
| 사이드바 ✕ | `term-kill` | **세션 종료** — 프로세스까지 내림(dev 서버도 죽음) |
| 칸 ✕ | 칸 비우기 | 세션 유지, 화면에서만 뺌 |

①②③④ 배지가 이 둘을 시각적으로 가른다. 이 구분이 무너지면 형이 dev 서버를 실수로 죽인다.

### D9. 복원

- localStorage 에는 **레이아웃·칸배치·활성 칸만**(`{layout, slots, focus}`). 세션 목록은 term-list 가 진실.
  사이드바 순서는 저장하지 않는다 — `created` 오름차순으로 파생하고, 그게 `셸 N` 라벨의 근거이기도 하다.
- 복원 시 대조해 죽은 tid 의 칸은 비운다. 다른 브라우저·창으로 열어도 같은 세션이 보인다.

## 데이터 흐름

```
term-list ──▶ 세션 스토어 ──▶ 사이드바 렌더
                  │
                  ├─▶ 멀티플렉스 SSE (tid=전체 살아있는 세션, from=오프셋맵)
                  │      └─▶ {tid,b64,off} ─▶ 인스턴스 있으면 xterm.write / 없으면 활동 닷
                  │
칸 배치(클릭·D&D) ─┴─▶ 인스턴스 최초 생성 → 그 tid만 from 없이 재연결(snap)
xterm.onData ────────▶ tid별 직렬 큐 ─▶ /api/term-input
ResizeObserver ──────▶ /api/term-resize (tid)
```

## 에러 처리

- `term-open` 실패(워크트리 없음 등) → `alert(e.message)`, 사이드바 항목 미생성(app-8-git 과 같은 패턴).
  `.git-err` 는 xterm 자체가 로드 실패한 경우에만 쓴다.
- SSE 끊김 → 지수 백오프 재연결(0.5s → 최대 5s, 2배씩), `from` 오프셋맵으로 이어받아 무중복.
- `exit` 이벤트 → 그 tid 인스턴스에 종료 문구 write, **칸은 유지**(헤더에 `(종료됨)`, 스크롤백 읽기 가능).
  형이 칸 ✕ 로 치울 때까지 남는다. **사이드바 항목은 사라진다** — `term_list` 가 `_reap_idle` 로 죽은 세션을
  아예 주지 않기 때문(시체 누수 방지가 그쪽 계약이다). 죽여도 안 죽는 ✕ 달린 유령 행보다 낫다.
- 그래서 **인스턴스 파기 = 죽었고(스토어에 없음) + 칸에도 안 떠 있을 때** — 매 렌더 스윕. D6 의 "세션 종료 시
  파기"를 "칸은 유지"와 화해시키는 유일한 해석이다. 이게 없으면 xterm(5000줄)+ResizeObserver 가 샌다.
- `term-input` 이 "세션 없음"(만료/재시작) → 스토어 갱신 후 그 세션을 죽은 것으로 표시.
- 링버퍼 갭(`from < base`) → snap 폴백(터미널 reset 후 재작성).

## 테스트 (`plugin/tests/test-term.sh` 갱신 + 추가)

계약이 바뀌므로 기존 단언을 **갱신**한다(신규 추가가 아니라 교체):

- ~~"워크트리당 세션 1개 재사용"~~ → **"셸은 매번 새 세션"**(같은 root 두 번 open → 다른 tid).
- **"에이전트는 재사용"** 은 유지(현행 d6/d7 단언 그대로).
- `term_list` 계약: 열린 세션이 tid·root·agent·created 와 함께 **created 오름차순으로** 나오고(라벨 `셸 N` 의 근거),
  kill 후 목록에서 빠진다. 정렬 단언은 값이 아니라 **반환 리스트의 위치**를 봐야 한다 — tid 로 키를 잡은 dict 를
  비교하면 정렬이 역순이어도 통과하는 공허한 단언이 된다.
- 멀티플렉스: 두 tid 동시 구독 → 각 이벤트에 올바른 tid 태그.
- `from` 재개: 오프셋 이후만 오고 중복 없음.

- 프론트 문법 계약 갱신: `termSendChain` grep → **tid별 큐**로 대상 변경, `app-10b-term-io.js` 로드 확인,
