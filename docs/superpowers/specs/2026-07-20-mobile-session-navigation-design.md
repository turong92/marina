# 모바일 세션 탐색 개편 설계

## 목표

폰에서 Marina를 열었을 때 프로젝트를 먼저 고르고, 해당 프로젝트의 Codex, Claude, Terminal 세션을 구분해 기존 작업에 바로 붙는다. 상단 조작부는 한 줄을 넘지 않고 세션 카드의 제목, 상태, 최근 작업이 서로 겹치지 않아야 한다.

## 화면 구조

- 상단 바: 목록에서는 `Marina`와 메뉴 버튼만 표시한다. 채팅에서는 왼쪽에 뒤로가기 아이콘을 추가한다.
- 메뉴: 새로고침, 알림, 로그아웃을 드롭다운에 모은다. 바깥을 누르거나 기능을 실행하면 닫힌다.
- 프로젝트 선택: 검색 위의 가로 스크롤 칩으로 프로젝트를 전환한다. 선택은 브라우저에 저장한다.
- 소스 선택: `전체`, `Codex`, `Claude`, `Terminal` 탭으로 현재 프로젝트의 세션을 필터링한다.
- 세션 목록: 전체 탭에서는 소스별 그룹으로 나누고, 각 카드에 소스 배지와 최근 작업을 표시한다. 임시 agent runner 터미널은 기존 서버 필터를 유지해 중복 노출하지 않는다.
- 빈 상태: 프로젝트나 소스에 세션이 없으면 현재 필터를 설명하는 한 줄만 표시한다.

## 상태와 데이터

- 새 서버 API를 만들지 않는다. 기존 `state.worktrees`, `state.sessions`, 세션의 `root`, `kind`, `target`을 조합한다.
- 프로젝트 키는 worktree의 `projectId`, `projectLabel`, `alias`, `root` 순서로 안정적으로 결정한다.
- 소스는 agent target의 `source`를 우선하고, 일반 PTY는 `terminal`로 분류한다.
- 자동 갱신 중에도 선택한 프로젝트, 소스, 세션을 유지한다. 사라진 프로젝트만 첫 번째 유효 프로젝트로 보정한다.

## 시각 원칙

- 기존 Marina의 푸른 강조색은 유지하되 배경, 구분선, 소스 배지에 중립색과 Codex/Claude 구분색을 사용한다.
- 카드는 고정 높이 대신 내용 기반 최소 높이를 사용하고 제목/미리보기 줄 수를 제한해 겹침을 방지한다.
- 360px 폭에서 상단은 44px 안팎, 터치 대상은 최소 40px로 유지한다.
- 다크 모드와 키보드 포커스를 함께 지원한다.

## 검증

- HTML 계약 테스트: 메뉴, 프로젝트 탭, 소스 탭, 그룹, 상태 저장 키가 존재한다.
- 기존 모바일 전송/로그인/자동 갱신 테스트가 모두 통과한다.
- 실제 3900 대시보드를 390x844 뷰포트로 열어 겹침, 메뉴 동작, 프로젝트/소스 전환, 기존 세션 진입을 확인한다.
