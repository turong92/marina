# 모바일 네이티브 채팅 설계

## 목표

폰에서 기존 Claude 또는 Codex 세션에 붙었을 때 데스크톱 클라이언트와 같은 언어로 메시지, 스킬, 파일, 서브에이전트를 다룬다. Marina 고유 호출 문법을 만들지 않고 선택한 세션의 원래 문법과 개념을 유지한다.

## 채팅 화면

- `최근 작업` 패널을 제거하고 대화 턴만 시간 순서대로 표시한다.
- 사용자 메시지는 오른쪽, 에이전트 메시지는 왼쪽에 배치하고 `assistant`, `user` 역할명은 노출하지 않는다.
- 일반 터미널 세션은 대화 턴이 없으므로 현재 출력을 마지막 타임라인 항목으로 표시한다.
- URL과 Markdown 링크는 새 탭에서 열 수 있게 하되 HTML은 실행하지 않는다.
- 사용자가 타임라인 끝에 있을 때만 새 응답에 맞춰 자동 스크롤한다. 위쪽을 읽고 있으면 `새 메시지` 버튼으로 이동한다.

## 네이티브 입력

- Claude 세션은 `/`로 명령과 스킬, `@`로 파일과 에이전트를 검색한다. 에이전트 선택은 Claude의 `@agent-<name>` 표현으로 입력한다.
- Codex 세션은 `$`로 스킬, `@`로 파일을 검색한다. 서브에이전트 위임은 별도 Marina 문법 없이 자연어 요청을 사용한다.
- `/` 명령은 현재 Marina의 resume 기반 전송 경로에서 실제 동작을 보장할 수 있는 항목만 노출한다. TUI 내부 상태를 직접 조작해야 하는 명령은 흉내 내지 않는다.
- 선택 항목은 원본 문법을 텍스트에 삽입할 뿐이며, 최종 메시지는 기존 `/mobile/api/send` 계약으로 전송한다.
- 입력창은 내용에 맞춰 자동 확장되고, 하드웨어 키보드의 Enter는 전송, Shift+Enter는 줄바꿈으로 동작한다.
- 초안은 세션별로 저장한다. 전송 실패 시 텍스트를 유지하고 같은 메시지를 다시 보낼 수 있다.

## 카탈로그

- 서버는 선택한 worktree를 기준으로 사용자 및 프로젝트 범위의 활성 스킬, 에이전트, 참조 가능한 파일을 최소한으로 탐색한다.
- Claude와 Codex 카탈로그는 분리해 반환한다. 이름이 같아도 서로 합치지 않는다.
- 플러그인 캐시 전체를 무조건 노출하지 않는다. 활성 설정으로 확인할 수 없는 플러그인은 목록에서 제외한다.
- 파일 검색은 입력한 검색어가 있을 때 worktree 내부의 추적 파일만 제한된 개수로 조회한다.

## 서브에이전트

- 메인 타임라인에 도구 호출 로그를 섞지 않는다.
- 채팅 화면의 햄버거 메뉴에 `작업 에이전트 N`을 추가한다.
- 항목을 누르면 바텀시트에서 실행 중 또는 완료된 자식 작업을 보여준다.
- 자식 작업을 누르면 요청, 마지막 결과, 확인 가능한 자식 대화 턴을 읽기 전용으로 펼친다.
- Claude의 Agent/Task 기록과 Codex의 spawn/wait 기록을 공통 `{id, title, status, preview, turns}` 형태로 정규화한다.
- 민감정보 마스킹은 기존 agent transcript와 같은 규칙을 적용한다.

## 시각 및 상호작용

- 현재 Marina의 중립 배경, 푸른 동작색, Claude/Codex 구분색을 유지한다.
- 입력 자동완성은 입력창 바로 위에 붙는 작은 목록으로 표시하고 세션 소스에 맞는 항목만 노출한다.
- 바텀시트와 자동완성은 360px 폭에서도 화면을 넘지 않고, 터치 대상은 최소 40px를 유지한다.
- 링크, 메뉴, 자동완성은 키보드 포커스를 지원한다.

## 검증

- 파서 단위 테스트로 Claude/Codex 서브에이전트 기록을 정규화하고 마스킹하는지 확인한다.
- HTML 계약 테스트로 최근 작업 제거, 역할명 제거, 소스별 트리거, 세션별 초안, 링크 렌더링, 재시도, 서브에이전트 시트를 확인한다.
- 기존 로그인, 자동 갱신, 세션 선택, 전송 테스트를 모두 유지한다.
- 실제 모바일 뷰포트에서 대화 스크롤, 입력 자동완성, 링크, 메뉴, 바텀시트를 직접 검증한다.
