날짜: 2026-07-10

## 배경·목표

- 현행 대시보드는 기능은 갖췄지만 카드 디자인이 조악하고(테두리·상시 버튼·밀도 불균형) UI 깨짐이 있다.
- Orca(onorca.dev) 실스크린샷에서 디자인 문법을 차용하되, marina 는 IDE 가 아니라
  **worktree 별 풀스택 dev runtime 콘솔**이라는 정체성을 유지한다.
- 확장 여지: 우측을 로그 전용에서 **뷰 탭 워크스페이스**로 일반화 — 깃 그래프(1단계 기구현,
  upbeat-shtern 브랜치)와 추후 터미널이 탭으로 들어온다.

## 비목표

- 3-pane rail 신설(프로젝트는 헤더 스위처 유지 — 프로젝트=설정 경계, 화면=선택 프로젝트의 worktree 운영)
- 터미널 탭의 실제 구현(자리·탭 구조만 이번 범위, PTY 는 후속)
- Orca 식 에이전트 오케스트레이션/에디터/브라우저 내장
- 등록 위저드·compose 편집 등 보조 흐름의 재설계(위치만 보조 흐름으로 유지)

## 확정 결정

### D1. 골격 — 현행 2단 유지(A+)

- 좌: worktree 카드 리스트(aside) · 우: 워크스페이스(section). 헤더: 프로젝트 스위처 + mem + 테마 + ↻.
- 헤더에 선택 프로젝트 요약 1줄: `worktree 3 · 실행 2`.

### D2. 우측 = 뷰 탭 워크스페이스

- 탭: **로그 | 깃 | 터미널(추후, disabled 표기)**. 로그 탭 = 현행 로그 뷰어 이식(Server/Console·run 선택·필터·게이지 유지).
- 깃 탭 = 기구현 깃 그래프 1단계 통합(레인 그래프+diff 모달).
- 탭 컨텍스트(어느 worktree·서비스)는 좌측 카드/서비스 행 선택을 따라간다. 탭 우측에 컨텍스트 표시
  (`feat/login · db ▾ · run #3 ▾`).
- 새 뷰 추가가 "탭 하나 추가"가 되도록 워크스페이스를 view registry 로 구성(로그·깃이 같은 계약으로 등록).

### D3. 카드 — Orca 문법

- **무테두리**: 간격·배경 명도로만 구분. 선택 카드만 배경 승격, hover 시 약한 배경.
- **카드 헤더**: 좌측 상태 점(worktree 종합) + 브랜치명(흰 semibold) + 우측 정렬 메타(attach n/m · 경과시간).
- **SERVICES 접힘 섹션**: 대문자 마이크로 라벨 `▾ SERVICES (n)` + 서비스 행(상태점·이름·우측 모노 값).
- **액션은 hover 시에만** 카드 우상단(⏹ ↻ ✎ ⋯)·서비스 행 hover(▶/⏹·로그·⋯). 상시 노출 버튼 제거.
- **상시 노출 슬롯**(접힘과 무관): ① 에러 원인+다음 액션 1줄(`db: POSTGRES_PASSWORD 미설정 → env 열기 · 로그`)
  ② 대표 게이트웨이 URL(`login.mdc.localhost ↗`).
- compose 편집·위저드·links·프로필 등 설정성 기능은 카드 ⋯ 메뉴/모달 뒤 보조 흐름.

### D4. 접힘 요약 — 상태별 카운트 (형: "갯수가 늘 수 있으니 카운트")

- `▸ SERVICES (4) ▶ 2 · ⟳ 1 · ✕ 1` — 0 인 상태는 생략. 정지만 있으면 `■ 2`.
- 서비스 수와 무관하게 길이 일정. 개별 식별은 펼침 또는 에러 줄로.

### D5. 상태 팔레트 (형: "기동중도 주황")

| 상태 | 색 | 아이콘 | 비고 |
|---|---|---|---|
| running | 초록 #34c98e | ▶ | |
| starting | 주황 #f0a132 | ⟳ | 빌드·헬스체크 중 포함 (`building…`) |
| error | 빨강 #e5484d | ✕ | exit≠0·기동 실패 — 원인 줄 동반 |
| stopped | 회색 #5a5f6a | ■ | |
| external | 청록 #38b6c4 | ⇄ | IDE/터미널 직접 실행 감지(기존 '외부 :port') |
| degraded | 주황 점 + 원인 줄 | ⚠ | Dockerfile 부재 등 — 시작 불가 사유를 에러 줄 슬롯에 |

- worktree 종합 점: 전부 running=초록 · 하나라도 starting 또는 일부만 running=주황 · 하나라도 error=빨강 · 전부 stopped=회색.
- 라이트 테마: 동일 문법에서 명도만 반전(배경 #fff 계열·카드 구분은 회색 배경, 색상 토큰은 채도 유지). 시스템/라이트/다크 3모드 현행 유지.

### D6. 포트·주소 표기 — 내부→호스트 (형 확정, 실측 검증 완료)

- 서비스 행 우측 모노: `3000→55011` (컨테이너 내부→호스트 자동할당). starting 은 `building…`, error 는 `exit 1`.
- hover 툴팁: "컨테이너 3000 → 호스트 55011 (자동할당·재시작마다 변동)" + 서비스별 게이트웨이 도메인.
- 클릭 = 값 복사. 행 ⋯ 메뉴: [게이트웨이로 열기 `<wt>-<svc>.<proj>.localhost:3902`] · [호스트포트로 열기] · [로그].
- 카드 하단 대표 URL = primary(web) 게이트웨이 도메인, 클릭=브라우저 열기.
- **실측 근거(2026-07-10)**: 대표 도메인 `…017576.mdc-main.localhost:3902/ko` → 200(500KB, 타이틀 렌더),
  서비스별 도메인 `…-index-api`·`…-search-api` `/docs`·`/openapi.json` → 전부 200 (실 caddy·실 docker).
  즉 웹 열기·서버 콜 모두 현행 게이트웨이가 이미 지원 — UI 는 노출만 하면 된다.

### D7. 액션 — hover 클러스터 + 실행/정지 토글 1개 (형 확정)

배치 철학: **A(hover 클러스터, Orca 순수)** — 카드에 상시 버튼 없음(카드=상태 표시 전용).
액션은 카드 우상단·서비스 행 우측에 **hover 시(또는 카드 선택 중 상시)** 나타난다.
불가능한 액션은 숨김(비활성 버튼 없음).

버튼 문법 (카드·행 동일 — 스코프만 다름):

- **토글 1개 `▶/⏹`** — 실행/정지는 버튼 하나가 상태 따라 얼굴을 바꾼다.
  - 카드(전체 스코프): 하나라도 실행·기동중이면 `⏹`(전체 정지), 전부 정지면 `▶`(전체 시작).
  - 행(그 서비스): running→`⏹`, stopped/error→`▶` (에러 재시도 = `▶` 와 동일 — 별도 재시도 버튼 없음).
- **`↻` 재시작 — 실행 중일 때만** 토글 옆에 나타난다 (형: "재기동은 실행중일 때만").
- **`⋯` 메뉴** — 나머지 전부: 카드=✎ compose · attach · links · 프로필 · 위저드 / 행=게이트웨이로 열기 ·
  호스트포트로 열기 · 주소 복사. 행 hover 에는 `로그`(그 서비스 로그 탭 포커스)도 포함.
- **external 행**: `⏹` 하나(기존 외부 프로세스 SIGTERM 정지) + ⋯[호스트포트로 열기].
- "일부만 실행" 카드에서 나머지 시작은 카드 토글이 아니라 **각 행의 `▶`** 로 — 카드 토글의 의미를
  단순하게 유지(실행 있음=⏹).

busy/진행 중 규칙 (기존 async lifecycle 유지):
- 액션 실행 중엔 토글이 스피너(`⟳`)로 바뀌고 클릭=취소. 같은 스코프의 다른 lifecycle 액션은 잠금
  (중복 기동 방지 — 기존 busy 개념 준수).
- start/restart 는 비동기(기존 동작) — starting 상태로 전환돼 진행을 보여주고, 완료/실패 시 상태·원인 줄 갱신.
- 파괴적 확인 없음(현행 유지). 외부 프로세스 정지는 현행처럼 대상 표시.
- 발견성 보완: **선택된 카드는 hover 없이도 클러스터 상시 표시** — 터치/스크린샷에서도 액션이 보인다.

## 상태 모델 (백엔드 계약)

- `/api/…` 세션 payload 를 서비스별 `state: running|starting|error|stopped|external|degraded` +
  `detail`(포트·exit코드·원인 텍스트·다음 액션 종류) 로 정규화한다 — UI 가 문자열 조합을 추측하지 않게.
- 원인+다음 액션 줄은 백엔드가 `{reason, actions:[{kind: env|logs|compose|retry, label}]}` 로 내려준다.
  (기존 로그 tail·exit 감지에서 추출 — 신규 진단 엔진은 비범위, 있는 정보의 구조화만.)

## 안정화 (UI 깨짐 — 기능 문제로 취급)

- 긴 브랜치/서비스명 overflow(ellipsis + 툴팁), 카드/행 높이 고정.
- 버튼 텍스트 줄바꿈·겹침 제거(호버 액션화로 대부분 해소).
- 좁은 화면: aside 접기(현행 rail 토글 유지), 카드 최소폭 보장.
- 모달/drawer 가 본문을 밀거나 덮어 깨지는 경우 제거.
- 상태별 액션 노출은 D7 매트릭스를 따른다(불가능한 액션은 숨김 — 비활성 버튼 없음).

## 테스트·검증 계획

- 기존 대시보드 API 테스트 회귀(payload 정규화 추가분 단위 테스트).
- 상태 매핑 단위 테스트: compose ps/exit/빌드중 → state 판정.
- 실브라우저 검증: marina-preview(:3901) + Aside 로 카드 hover 액션·접힘 요약·탭 전환·게이트웨이 열기,
  다크/라이트, 좁은 화면. (내 선호 규약: 실측 스크린샷 확인.)
- 깃 탭: 기구현 깃 그래프의 기존 테스트 유지 + 탭 통합 스모크.

## 구현 순서 (플랜에서 상세화)

1. 상태 모델 정규화(백엔드 payload) — UI 와 독립 커밋
2. 카드 재설계(D3·D4·D5·D6) — 기존 기능 등가 유지
3. 워크스페이스 탭 셸(D2) + 로그 뷰 이식
4. 깃 그래프 탭 통합(upbeat-shtern 1단계 이식)
5. 안정화 체크리스트 + 실브라우저 검증
