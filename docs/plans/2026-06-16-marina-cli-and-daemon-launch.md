# marina CLI + robust dashboard launch — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a version-stable `marina` terminal command and a dashboard daemon that auto-restarts on macOS and Linux, both anchored on one self-resolving launcher so auto-update and auto-restart compose.

**Architecture:** A single resolver module (`marina-resolve.sh`) holds the logic that, given the harness's `installed_plugins.json`, prints the current plugin install path. Because the user shim (`~/.local/bin/marina`) and the daemon launcher (`~/.marina/dashboard-launch.sh`) live *outside* the versioned plugin dir, the resolver provides an **emit** function that writes a self-contained launcher to a target path. `install-cli` emits the user shim; `marina-dashboard.sh` emits the daemon launcher and points the OS supervisor (launchd / systemd-user / nohup) at it.

**Tech Stack:** bash + `/usr/bin/python3` (stdlib only). Tests are shell scripts under `plugin/tests/`, isolated via a temp `MARINA_HOME` / `CLAUDE_CONFIG_DIR`, following the existing `test-attach-clean-codex-worktree.sh` pattern.

**Scope (this plan):** shared resolver, `marina install-cli` / `uninstall-cli`, cross-platform daemon launch. Out of scope (2nd plan): `marina.sh infer`, slash commands, dashboard API/UI, README. Reference: `docs/specs/2026-06-15-register-ux-design.md` components 3 + 7.

**Resolver contract (the keystone — every task depends on it):**

```bash
# marina-resolve.sh — sourced by in-plugin scripts. NEVER sourced by emitted launchers
# (they live outside the plugin); instead they embed the resolution snippet below.

# Ordered manifest candidates: Claude first, then Codex (best-effort; verify on a Codex box).
marina_manifest_candidates() {
  printf '%s\n' \
    "${CLAUDE_CONFIG_DIR:-$HOME/.claude}/plugins/installed_plugins.json" \
    "${CODEX_HOME:-$HOME/.codex}/plugins/installed_plugins.json"
}

# Print current install path for marina@marina-dev, or non-zero if not found.
marina_install_path() {
  local mf
  while IFS= read -r mf; do
    [[ -f "$mf" ]] || continue
    python3 - "$mf" <<'PY' && return 0
import json, sys
try:
    d = json.load(open(sys.argv[1], encoding="utf-8"))
    print(d["plugins"]["marina@marina-dev"][0]["installPath"])
except Exception:
    sys.exit(1)
PY
  done < <(marina_manifest_candidates)
  return 1
}

# Write a self-contained launcher to $1. $2 = exec kind: "entrypoint" | "dashboard".
# The emitted file re-resolves the install path on every invocation (survives auto-update).
marina_emit_launcher() {
  local target="$1" kind="$2" exec_line
  case "$kind" in
    entrypoint) exec_line='exec "$ip/scripts/marina-entrypoint.sh" "$@"' ;;
    dashboard)  exec_line='exec "${MARINA_PYTHON:-$(command -v python3 || echo /usr/bin/python3)}" "$ip/scripts/marina-control.py"' ;;
    *) echo "marina_emit_launcher: bad kind: $kind" >&2; return 2 ;;
  esac
  mkdir -p "$(dirname "$target")"
  cat > "$target" <<EOF
#!/usr/bin/env bash
# AUTO-GENERATED by marina. Re-resolves the current plugin version on every run.
set -euo pipefail
ip="\$(python3 - <<'PY'
import json, os, sys
cands = [
    os.path.expanduser(os.environ.get("CLAUDE_CONFIG_DIR", "~/.claude") + "/plugins/installed_plugins.json"),
    os.path.expanduser(os.environ.get("CODEX_HOME", "~/.codex") + "/plugins/installed_plugins.json"),
]
for mf in cands:
    try:
        d = json.load(open(mf, encoding="utf-8"))
        print(d["plugins"]["marina@marina-dev"][0]["installPath"]); sys.exit(0)
    except Exception:
        continue
sys.exit(1)
PY
)" || { echo "marina: cannot locate installed plugin (is marina@marina-dev installed?)" >&2; exit 1; }
$exec_line
EOF
  chmod +x "$target"
}
```

### Task 3: Cross-platform daemon launch

**Files:**
- Modify: `plugin/scripts/marina-dashboard.sh`
- Test: `plugin/tests/test-dashboard-launch.sh`

**Changes to `marina-dashboard.sh`:**
1. Source the resolver: after the `*_SCRIPT` vars, add `source "$SCRIPT_DIR/marina-resolve.sh"`.
2. Add `LAUNCHER="$MARINA_HOME/dashboard-launch.sh"` and a `write_launcher()` that calls `marina_emit_launcher "$LAUNCHER" dashboard`.
3. Add a supervisor selector:
```bash
supervisor() {
  if [[ "$(uname -s)" == "Darwin" ]] && command -v launchctl >/dev/null 2>&1; then echo launchd
  elif command -v systemctl >/dev/null 2>&1 && systemctl --user show-environment >/dev/null 2>&1; then echo systemd
  else echo nohup; fi
}
```
4. `write_plist`: change `ProgramArguments` to `[$LAUNCHER]` instead of `[$PYTHON_BIN, $CONTROL_SCRIPT]`.
5. Add `write_systemd_unit()` writing `~/.config/systemd/user/marina-dashboard.service` with `ExecStart=$LAUNCHER`, `Restart=on-failure`, `[Install] WantedBy=default.target`.
6. Rework `start()` to: `write_launcher`; then branch on `supervisor`:
   - `launchd` -> existing bootstrap/kickstart path (plist now points at `$LAUNCHER`).
   - `systemd` -> `loginctl enable-linger "$USER" 2>/dev/null || true`; `systemctl --user daemon-reload`; `systemctl --user enable --now marina-dashboard`.
   - `nohup` -> existing nohup fallback **plus** `echo "warn: no launchd/systemd — auto-restart NOT configured" >&2`.
7. `stop()`: also handle `systemctl --user disable --now marina-dashboard` when systemd unit exists.
8. **Test hook:** if `MARINA_DRY_RUN=1`, `start()` writes the launcher + plist/unit and prints the chosen supervisor, but does **not** bootstrap/enable/exec. (Lets tests assert generation without launching anything.)
